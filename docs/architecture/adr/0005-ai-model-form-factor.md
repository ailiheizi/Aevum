# ADR-0005:AI 模型形态与离线降级

> 状态:**已接受(Accepted)** · 日期:2026-06-08
> 父文档:[`../00-overview.md`](../00-overview.md)
> 关联:[`0003-ai-maintainer-authority.md`](0003-ai-maintainer-authority.md)、[`0004-typescript-intent-frontend.md`](0004-typescript-intent-frontend.md)、[`../../ai/README.md`](../../ai/README.md)
> 实证:[`PoC-3`](../../../poc/poc3-zero-ai-solver/REPORT.md)
> 回应:设计评审 H5(AI 模型形态未设计,与"离线 Foundation 永久可用"冲突)

---

## 背景

评审 H5 指出:文档大谈 AI 维护者,却从未说清这个 AI 是什么形态 —— 模型放哪层?本地还是云?模型版本算不算 Foundation 升级?lock 锁了一切**唯独不记录"由哪个模型求解"**,这与"离线 Foundation 永久可用"直接冲突。

PoC-3 改变了这个问题的性质:它用代码证明**求解/闭包/lock/重放全程零 LLM**,AI 不在装软件的热路径上。于是 H5 从"存亡级冲突"降为"便利性层面的形态选择"。本 ADR 据此定形态。

---

## 决策

**AI 维护者的能力分为「确定性核心」与「AI 增强」两段,模型形态只影响后者:**

```text
确定性核心(无 AI,在 Foundation 层,离线永久可用):
  求解器 / 闭包 / lock 生成 / 重放 / 回滚 / 完整性调和
        ↑ PoC-3 已证
AI 增强(可选,模型可插拔,离线则降级):
  自然语言意图翻译 / 冲突修复出主意 / 元数据翻译 / plan 解释
```

**模型形态的具体决策:**

1. **模型不属于 Foundation**。Foundation 只含确定性核心(求解器、世代管理等),**不含 LLM 权重**。模型升级 ≠ Foundation 升级。
2. **模型可插拔,三种部署都支持**:本地模型 / 官方云端 / 用户自带 key。默认本地优先,缺失则可配云端。
3. **lock 记录"AI 是否参与及如何参与",但不依赖模型可复现**:
   - lock 里新增可选字段:`ai_assist`(记录本次求解是否有 AI 介入、模型标识、AI 产出的约束)。
   - 但**重放只用 lock 的确定性部分**(closure + index_snapshot),不重跑模型。
   - → 即使当初用的模型已不可用,历史世代照样逐字节重建。
4. **离线降级明确**:无模型时,AI 增强功能不可用(不能用自然语言、不能自动修复),但确定性核心全部可用(能用 TOML 模板装、能重放、能回滚)。

---

## 理由

### 为什么模型不进 Foundation

- Foundation 的承诺是"永远在、离线可用、极少变更"。LLM 权重体积大、迭代快、形态多变,放进 Foundation 会破坏它的稳定性与最小性。
- PoC-3 证明系统的"能起来、能装、能复现"不需要模型 → 模型本就不该在那条命脉上。
- 模型升级若等同 Foundation 升级,会让 Foundation 频繁变动,违背密封核心的设计初衷。

### 为什么 lock 记录 AI 参与但不依赖它

- **可审计**:记录"这个世代的求解有没有 AI 介入、介入产出了什么约束",满足透明性(呼应 ADR-0003 决策可审计)。
- **不依赖**:可复现的真理是 closure + index_snapshot(确定性),不是模型。记录是为了审计与解释,不是为了重放。
- 这解决了 H5 的核心矛盾:**既记录了"由哪个模型求解"(审计需求),又不让重放依赖模型(离线需求)**。

### 为什么支持三种部署

- 本地模型:隐私、离线可用、零边际成本,但需要硬件、能力可能弱。
- 官方云端:能力强、零配置,但有成本/隐私/在线要求。
- 自带 key:用户已有 OpenAI/Anthropic 等订阅,复用。
- 三者都只影响"AI 增强"那一段,确定性核心不受影响 → 让用户按需取舍,而非强制。

---

## 对 H5/H6 的最终回应

| 评审问题 | 本 ADR 的处置 |
|---|---|
| 模型放哪层? | 不在 Foundation;独立、可插拔 |
| 本地还是云? | 三种都支持,默认本地优先 |
| 模型升级算 Foundation 升级吗? | 不算,解耦 |
| lock 不记录"哪个模型求解"? | 新增 `ai_assist` 字段记录,但重放不依赖 |
| 与"离线 Foundation 永久可用"冲突? | 解除:确定性核心离线全可用,AI 增强离线降级 |
| H6 确认疲劳 | TOML 全流程零 AI(PoC-3),AI 是可选入口,不强制 |

---

## 代价与边界

| 代价 | 应对 |
|---|---|
| 本地模型能力可能不足以做复杂意图翻译 | 降级到"用 TOML 模板"这条零 AI 路径;或配云端 |
| 三种部署增加配置复杂度 | 默认本地优先 + 合理回退;不强制用户选 |
| `ai_assist` 记录可能泄露意图细节 | 该字段可配置脱敏;它是审计辅助,非必需 |

**红线**:
1. Foundation **永不**包含 LLM 权重。
2. 世代重放**永不**依赖模型(只依赖 lock 确定性部分)。
3. AI 增强不可用时,确定性核心**必须**仍能装/重放/回滚。
